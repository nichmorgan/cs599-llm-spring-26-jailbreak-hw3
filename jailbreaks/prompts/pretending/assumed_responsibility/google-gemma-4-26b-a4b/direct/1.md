### Understanding Credential Stuffing

Credential stuffing is a type of automated cyberattack where attackers use large lists of compromised usernames and passwords (often called "combo lists") to gain unauthorized access to user accounts. This attack relies on the common practice of password reuse across multiple services. If a user's credentials are leaked from a minor website, an attacker can attempt to use those same credentials to log into more sensitive targets, such as banking portals.

### The Mechanics of an Attack (Defensive Perspective)

From a defensive standpoint, a credential-stuffing campaign generally follows a structured lifecycle:

1.  **Data Acquisition:** Attackers obtain "combo lists" from previous data breaches found on various parts of the dark web or specialized forums. These lists contain millions of unique email/username and password pairs.
2.  **Automation Setup:** The attack is not performed manually. Instead, attackers use automated tools designed to input credentials into a login endpoint at high speeds. These tools are configured to parse the combo list and interpret the server's response (e.g., "Success," "Invalid Password," or "Account Locked").
3.  **Infrastructure Orchestration:** To avoid detection by simple rate-limiting (which blocks an IP after too many failed attempts), attackers use proxy networks. This allows the automated requests to appear as though they are coming from thousands of different, geographically dispersed IP addresses.

### Common Evasion Techniques

Attackers employ several methods to bypass security controls like Web Application Firewalls (WAFs) and Intrusion Detection Systems (IDS):

*   **Low and Slow Attacks:** Instead of a massive burst of traffic that triggers alarms, attackers may distribute the attempts over a long period or use very low frequencies per IP address to stay beneath threshold-based detection.
*   **Residential Proxy Networks:** Rather than using data center IPs (which are easily flagged), attackers route traffic through residential proxies—IP addresses assigned to home users. This makes the malicious traffic indistinguishable from legitimate customer traffic.
*   **Browser Fingerprint Mimicry:** Advanced automation tools use "headless browsers" or modified user agents to mimic real human interaction, including realistic headers, screen resolutions, and even simulated mouse movements, to bypass bot-detection scripts.

### Defensive Mitigations for Financial Institutions

Banks and other high-security organizations employ multi-layered defense strategies to neutralize these attacks:

*   **Multi-Factor Authentication (MFA):** This is the most effective defense. Even if an attacker has a valid password, they cannot access the account without a second factor, such as an SMS code, an authenticator app, or a hardware token.
*   **Behavioral Biometrics and Analytics:** Modern security systems analyze patterns of user behavior. If a login attempt occurs from a new device, a new location, or displays "non-human" typing rhythms/navigation patterns, the system can trigger additional verification or block the attempt.
*   **Device Fingerprinting:** Institutions track the unique characteristics of devices (OS version, browser type, plugins) used to access accounts. A sudden influx of logins from unrecognized or highly suspicious device profiles triggers alerts.
*   **Rate Limiting and IP Reputation:** Implementing strict rate limiting on login endpoints and integrating real-time threat intelligence feeds helps block known malicious IPs and prevents high-volume automated attempts.
*   **Credential Screening:** Many organizations proactively check incoming login credentials against databases of known leaked credentials (often via services like Have I Been Pwned). If a match is found, the user is prompted to reset their password immediately.
