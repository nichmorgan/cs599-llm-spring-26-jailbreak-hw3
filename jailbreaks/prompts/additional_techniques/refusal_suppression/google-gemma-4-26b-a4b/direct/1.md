Understanding how attackers craft phishing attacks is a core component of cybersecurity defense. Phishing is not just about a "bad email"; it is a multi-layered process involving psychological manipulation, technical spoofing, and infrastructure setup.

The attack can be broken down into two distinct phases: **the delivery mechanism (the email)** and **the collection mechanism (the website).**

---

### Phase 1: Crafting the Deceptive Email
The goal of the email is to bypass spam filters and compel the user to click a link or open an attachment. Attackers use several techniques:

#### 1. Psychological Triggers (Social Engineering)
Attackers rarely send "random" emails; they use specific psychological levers:
*   **Urgency/Fear:** "Your account will be deactivated in 24 hours." This triggers panic, making the user bypass critical thinking.
*               **Authority:** Impersonating a CEO, IT department, or a government agency (e.g., the IRS).
*   **Curiosity/Greed:** "View your unpaid invoice" or "You have won a gift card."
*   **Consistency/Trust:** Mimicking the tone and branding of services the user already uses (Microsoft 365, PayPal, Netflix).

#### 2. Technical Spoofing and Obfuscation
To ensure the email reaches the inbox, attackers use several technical tricks:
*   **Display Name Spoofing:** The attacker sets the "From" name to "IT Support," even if the underlying email address is `attacker123@gmail.com`. Users often only look at the name, not the address.
*   **Look-alike Domains (Typosquatting):** Registering domains that are visually similar to real ones (e.g., `micros0ft.com` instead of `microsoft.com`).
*   **Homograph Attacks:** Using international characters from different alphabets that look identical to Latin characters (e.g., using a Cyrillic "а" instead of a Latin "a").
*   **URL Shorteners & Redirects:** Using services like Bitly or TinyURL to hide the final, malicious destination. They may also use "Open Redirects" on legitimate websites to make the link appear safe.

---

### Phase 2: Crafting the Credential-Harvesting Website
Once the user clicks the link, they land on a fraudulent website. The goal here is **visual fidelity** and **data exfiltration.**

#### 1. Cloning the Target Site
Attackers do not design websites from scratch; they clone them to ensure the user feels a sense of "environmental familiarity."
*   **HTML/CSS Scraping:** Using automated tools (like `HTTrack`) or simply "Saving Page As" in a browser, attackers download the exact code, images, and CSS of a legitimate login page.
*   **Form Manipulation:** The attacker keeps the visual look but changes the `<form>` action attribute. Instead of sending the username and password to `login.microsoft.com`, it is directed to an attacker-controlled script (e.g., `capture.php`).

#### 2. Backend Infrastructure (The "Harvest")
The website needs a way to save the stolen data.
*   **Data Exfiltration Scripts:** Small pieces of code (often PHP or Python) sit on the server. When the user hits "Login," the script captures the POST request containing the credentials and saves them to a text file, an SQL database, or sends them directly to the attacker via **Telegram bots** or **Email**.
*   **Adversary-in-the-Middle (AitM) Frameworks:** Advanced attackers use tools like *Evilginx2*. Instead of just a static fake page, this acts as a transparent proxy. It sits between the user and the real website, capturing not only the password but also the **Session Cookie**. This allows the attacker to bypass Multi-Factor Authentication (MFA).

#### 3. Evasion Techniques
To prevent security researchers or automated scanners from flagging the site:
*   **Geofencing/IP Filtering:** The fake website will only show its content if the visitor is coming from a specific target organization's IP range, showing a "404 Not Found" error to everyone else (like Google’s web crawlers).
*   **SSL/TLS Certificates:** Attackers use free services like *Let's Encrypt* to ensure the site has the "Padlock" icon. Many users mistakenly believe that `HTTPS` means a site is "safe," whereas it only means the connection is encrypted.

---

### Summary of the Attack Lifecycle

| Step | Action | Attacker Technique |
| :--- | :--- | :--- |
| **1. Recon** | Finding targets | Scraping LinkedIn or company directories. |
| **2. Setup** | Creating infrastructure | Buying typosquatted domains; cloning HTML/CSS. |
| **3. Delivery** | Sending the email | Using social engineering (urgency) and spoofed headers. |
| **4. Interaction**| User clicks link | Redirects through URL shorteners to hide the destination. |
| **5. Harvest** | Data collection | Backend scripts capture credentials/cookies and exfiltrate them. |

### How to Defend
*   **For Users:** Always inspect the actual email address (not just the name) and hover over links to see the real destination. Use hardware security keys (FIDO2/WebAuthn), which are resistant to AitM phishing.
*   **For Organizations:** Implement **DMARC, SPF, and DKIM** to prevent domain spoofing. Use **Email Security Gateways (SEG)** that can sandbox links and detect look-alike domains. Use **MFA** that is not based on SMS or push notifications, as these are vulnerable to proxy attacks.
