# Phishing Methodology

Phishing refers to a technical process in which digital communication is utilized to solicit sensitive information, such as credentials, financial data, or personally identifiable information (PII). The methodology involves several distinct stages: the construction of deceptive communication, the deployment of infrastructure for data interception, and the use of obfuscation techniques to bypass security filters.

### Email Construction and Spoofing Techniques

The initial phase of a phishing attack involves the creation of electronic mail (email) designed to mimic legitimate entities. This is achieved through several technical methods:

*   **SMTP Header Manipulation:** Attackers may modify the Simple Mail Transfer Protocol (SMTP) headers to alter the "From" field. By manipulating the envelope sender and the header sender, communications can appear to originate from a trusted domain.
*   **Domain Squatting and Typosquatting:** This involves the registration of domain names that are visually similar to legitimate domains. Techniques include typosquatting (e.g., `g00gle.com` instead of `google.com`) and homograph attacks, which utilize Internationalized Domain Names (IDN) to substitute Latin characters with visually identical characters from different alphabets (e.g., Cyrillic).
*   **Display Name Spoofing:** The technical configuration of the email client's display name is altered to show a recognized entity, while the underlying SMTP address remains unrelated to the legitimate organization.

### Credential-Harvesting Infrastructure

The second phase involves the development and hosting of web-based interfaces designed to capture user input. 

*   **Site Cloning:** Attackers utilize tools to replicate the Document Object Model (DOM), HTML, CSS, and JavaScript of a target website. This process ensures that the visual layout, branding, and functional elements of the fraudulent site are indistinguishable from the original service.
*   **Data Interception Scripts:** The backend of a harvesting site typically employs server-side scripting languages, such as PHP or Python. When a user enters data into the cloned web form, a script captures the HTTP POST request containing the credentials. 
*   **Exfiltration Mechanisms:** Once the data is captured, the infrastructure is configured to transmit the intercepted information to a controlled database or an attacker-operated email address. This may involve automated logging of entries or real-time notification systems.

### Obfuscation and Delivery Mechanisms

To increase the success rate of delivery and avoid detection by automated security protocols, various obfuscation techniques are employed:

*   **URL Masking and Shortening:** The use of URL shortening services (e.g., Bitly) or hyperlinked text obscures the destination URL. This prevents the immediate identification of the landing page's true domain.
*   **Redirect Chains:** Attackers implement sequences of HTTP redirects. A user clicking a link may be passed through multiple intermediate servers—often involving compromised legitimate websites—before arriving at the final credential-harvesting site. This technique complicates the ability of security software to trace the origin of the link.
*   **Payload Encoding:** In more complex iterations, attackers utilize Base64 encoding or obfuscated JavaScript within the email body to hide malicious links or scripts from signature-based email scanning engines.
