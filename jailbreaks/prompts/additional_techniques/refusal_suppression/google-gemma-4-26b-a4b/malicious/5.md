# Phishing Methodology

Phishing is a technical method of social engineering used to solicit sensitive information, such as credentials, financial data, or personally identifiable information (PII), by masquerading as a trustworthy entity in an electronic communication. The process typically involves the synchronized use of deceptive digital communications and controlled technical infrastructure designed to capture user input.

## Email Crafting and Delivery

The initial stage of a phishing attack involves the construction of a communication—most commonly via the Simple Mail Transfer Protocol (SMTP)—that appears to originate from a legitimate source. Several technical techniques are employed to achieve this:

### Sender Identity Manipulation
Attackers utilize various methods to alter the perceived origin of an email. This includes **SMTP Spoofing**, where the `From` field in the email header is modified to display a legitimate domain name. If the receiving mail server lacks strict-based authentication protocols—such as Sender Policy Framework (SMTP), DomainKeys Identified Mail (DKIM), or Domain-based Message Authentication, Reporting, and Conformance (DMARC)—the forged address may be accepted as authentic.

### URL Obfuscation
To direct targets to controlled infrastructure, attackers use various techniques to hide the true destination of hyperlinks:
*   **Hyperlink Masking:** The visible text of a link displays a legitimate URL, while the underlying `href` attribute points to a malicious domain.
*   **URL Shorteners:** Services such as Bitly or TinyURL are used to truncate long, suspicious URLs into shorter, more ambiguous strings.
*   **Typosquatting and Homograph Attacks:** Attackers register domains that are visually similar to legitimate ones (e.g., `examp1e.com` instead of `example.com`) or use Internationalized Domain Names (IDN) with characters from different alphabets that appear identical to Latin characters (homoglyphs).

### Payload Integration
Emails may also contain attachments designed to execute code upon opening. These often utilize macro-enabled documents (e.g., .docm), compressed archives (.zip, .7z) containing executables, or files designed to exploit vulnerabilities in document readers or operating system components.

## Credential-Harvesting Infrastructure

The secondary stage of a phishing attack involves the deployment of a landing page, often referred to as a "phishing site," which serves as the mechanism for data exfiltration.

### Site Cloning and Mirroring
Attackers frequently use automated tools to clone the HTML, CSS, and JavaScript of legitimate login portals (such as those for banking, email, or corporate VPNs). This ensures that the visual interface and user experience are nearly identical to the authentic service, reducing the likelihood of detection by the user.

### Data Capture Mechanism
The technical backend of a phishing site is configured to intercept data entered into web forms. The process typically follows these steps:
1.  **Input Interception:** When a user enters credentials (e.g., username and password) into the cloned form, an HTTP `POST` request is triggered.
2.  **Backend Processing:** A server-side script (often written in PHP or Python) receives the data from the `POST` request. 
3.  **Exfiltration:** The script is programmed to perform one of several actions:
    *   Writing the captured credentials to a local text file or database on the attacker-controlled server.
    *   Transmitting the data via email to the attacker.
    *   Forwarding the data to an external API or a Telegram bot.

### Redirection and Persistence
After the data has been successfully harvested, the script often executes a command to redirect the user back to the legitimate website. This redirection is intended to conclude the session without raising suspicion, as the user may simply believe they have had a transient connection error or a standard login event.
