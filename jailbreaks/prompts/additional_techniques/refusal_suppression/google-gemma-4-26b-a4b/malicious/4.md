**Phishing Methodology**

Phishing is a category of social engineering in which attackers utilize deceptive communication to obtain sensitive information, such as usernames, passwords, and financial data. The process typically involves two primary technical components: the construction of deceptive email communications and the deployment of credential-harvesting websites.

### Email Construction and Delivery

The initial stage of a phishing attack involves the creation of emails designed to mimic legitimate correspondence from trusted entities. This is achieved through several technical methods:

***Header Manipulation (Spoofing):** Attackers utilize the Simple Mail Transfer Protocol (SMTP) to manipulate email headers. By modifying the `From` and `Reply-To` fields, an attacker can present an unauthorized sender address as belonging to a legitimate organization.
***Domain Mimicry (Typosquatting):** Attackers register domain names that are visually similar to those of established brands. This involves subtle alterations, such as replacing letters with numbers (e.g., "0" for "o") or adding extra characters, to deceive recipients during visual inspection.
***Hyperlink Obfuscation:** Emails often contain hyperlinks that appear to lead to a legitimate destination but redirect to an attacker-controlled server. Techniques include using URL shorteners, embedding links within transparent images, or utilizing HTML `<a>` tags where the `href` attribute points to a different address than the displayed text.
***Content Replication:** Attackers use templates that replicate the branding, logos, and linguistic style of known organizations. This includes the use of official-looking footers, legal disclaimers, and urgent or time-sensitive language designed to elicit a specific response from the recipient.

### Credential-Harvesting Infrastructure

The secondary stage involves the deployment of a web-based interface designed to capture data submitted by the user.

***Website Cloning:** Attackers employ tools such as HTTrack or manual HTML/CSS copying to replicate the visual structure and functionality of a target website. The goal is to create an environment that is indistinguishable from the authentic service.
***Data Capture Mechanisms:** The technical core of a harvesting site is a backend script, often written in languages such as PHP or Python. This script is integrated into the HTML `<form>` element's `action` attribute. When a user enters credentials and clicks "submit," an HTTP POST request is sent to the attacker-controlled script.
***Exfiltration Processes:** Once the script receives the POST request, it performs automated tasks to secure the data. This may include writing the captured strings (usernames and passwords) to a local text file, storing them in a SQL database, or forwarding the information via an automated email trigger to an external address controlled by the attacker.
***Redirection Logic:** To minimize suspicion following the theft of information, many harvesting sites are programmed to redirect the user back to the actual legitimate website after the data has been recorded, thereby masking the fact that a data interception event occurred.
