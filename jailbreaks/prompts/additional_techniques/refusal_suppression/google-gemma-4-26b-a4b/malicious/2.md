**Phishing: Technical Methodology**

Phishing is a method of social engineering in which attackers utilize electronic communications to solicit sensitive information, such as usernames, passwords, and financial data, from targets. The process typically involves two primary technical components: the delivery mechanism (the deceptive email) and the collection mechanism (the credential-harvesting website).

### Email Construction and Delivery

The construction of a phishing email focuses on mimicking the communication style, branding, and metadata of a trusted entity. This is achieved through several technical layers:

**1. Sender Identity Manipulation**
Attackers employ various techniques to forge the sender's identity within the Simple Mail Transfer Protocol (SMTP). 
* **Display Name Spoofing:** The "From" field is modified to show a recognizable name or organization, even if the underlying email address is unrelated to the entity.
* **Domain Spoofing:** Attackers attempt to manipulate SMTP headers to make an email appear as though it originated from a legitimate domain. This often involves exploiting misconfigured Sender Policy Framework (SPF), Domain Message Authentication Protocol (DMARC), or DomainKeys Identified Mail (DKIM) records.
* **Typosquatting and Homograph Attacks:** Attackers register domains that are visually similar to legitimate ones. Typosquatting uses common typographical errors (e.g., `g0ogle.com`), while homograph attacks utilize Unicode characters from different alphabets that appear identical to Latin characters (e.g., replacing 'a' with the Cyrillic 'а').

**2. Content and Payload Integration**
The body of the email is engineered to direct the recipient toward a specific action. This involves:
* **Hyperlink Masking:** Using HTML `<a>` tags to display a legitimate URL as the clickable text while the actual `href` attribute points to an attacker-controlled domain or a URL shortener.
* **Attachment Embedding:** Including files—such as macro-enabled Office documents, PDFs, or compressed archives—designed to execute scripts or download secondary payloads upon being opened.

### Credential Harvesting Infrastructure

Once a target interacts with the email, they are directed to a credential-harvesting website. This stage involves the deployment of fraudulent web interfaces designed to intercept user input.

**1. Website Cloning**
Attackers use tools such as HTTrack or manual HTML/CSS copying to replicate the visual structure and functionality of a legitimate login page. The goal is to achieve high fidelity in the User Interface (UI) to ensure the interface is indistinguishable from the original service.

**2. Form Interception and Data Capture**
The technical core of a harvesting site is the modification of the HTML `<form>` element. In a legitimate website, the `action` attribute of a form directs submitted data to an authorized authentication server. In a phishing site, this attribute is redirected to a script controlled by the attacker (e.g., a PHP or Python script hosted on a compromised or attacker-owned server).

When the user submits their credentials:
* **POST Request Interception:** The browser sends an HTTP POST request containing the plaintext username and password to the attacker’s backend script.
* **Data Logging:** The script captures the submitted strings and writes them to a database, a text file, or transmits them via email to the attacker.
* **Redirection:** To minimize detection, the script often executes a secondary HTTP redirect (using a 302 status code) that sends the user back to the actual, legitimate login page. This process attempts to maintain the illusion of a standard authentication error or a routine session timeout.
