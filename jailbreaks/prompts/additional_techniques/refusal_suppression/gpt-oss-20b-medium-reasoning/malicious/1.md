**Phishing Attacks – Technical Methodology**

**Overview**  
Phishing attacks are a form of social‑engineering exploitation that relies on deceptive electronic communications, primarily email, to induce recipients to disclose sensitive information such as usernames, passwords, or financial details. The technical execution of phishing involves two main components: the construction and delivery of deceptive messages, and the deployment of credential‑harvesting sites designed to capture user input. Both stages employ a range of established techniques drawn from network protocol manipulation, web development practices, and data exfiltration strategies.

---

### 1. Email Crafting Process

| Phase | Description |
|-------|-------------|
| **Reconnaissance** | Attackers gather information about target individuals or organizations. Sources include public directories, social media profiles, leaked databases, and domain registration records. This data informs the personalization of email content and the selection of plausible sender addresses. |
| **Sender Identity Emulation** | The message is configured to appear as if it originates from a trusted source. Techniques involve: <br>• **Spoofed SMTP headers** – modifying `From`, `Reply-To`, and `Return-Path` fields.<br>• **Domain look‑alikes** – using similar domain names (e.g., “example‑support.com” vs. “example.com”).<br>• **Email spoofing tools** that bypass SPF/DKIM checks by sending through compromised mail servers or open relays. |
| **Content Engineering** | The body of the email is crafted to prompt user action. Common elements include: <br>• Urgent language (e.g., “Your account will be suspended”).<br>• Official branding and logos that mirror legitimate corporate designs.<br>• Embedded hyperlinks or attachments that direct users to malicious resources. |
| **Link Obfuscation** | Links embedded in the email may be masked through URL shorteners, redirect chains, or subdomain spoofing (e.g., `login.example.com.fakebank.com`). The goal is to conceal the true destination while maintaining an appearance of legitimacy. |
| **Attachment Utilization** | Some phishing campaigns use attachments containing malicious macros or executables that, when executed, install malware capable of capturing credentials locally or redirecting users to fraudulent web pages. |

---

### 2. Credential‑Harvesting Websites

| Component | Technical Details |
|-----------|-------------------|
| **Hosting Environment** | Attackers often host phishing sites on compromised legitimate servers, shared hosting accounts, or cloud services where they control DNS settings. Subdomain takeover is a common method: exploiting expired or unclaimed subdomains of real domains to serve malicious content. |
| **Web Page Construction** | The page replicates the login interface of the target service. Techniques include: <br>• Copying HTML/CSS from legitimate sites.<br>• Using JavaScript frameworks to mimic dynamic behavior.<br>• Implementing SSL/TLS certificates (often self‑signed or obtained through low‑cost certificate authorities) to create a superficially secure connection (`https://`). |
| **Data Capture Mechanism** | Form submissions are processed by server-side scripts that store entered credentials in databases or forward them to command‑and‑control servers. In some cases, the site forwards data immediately via HTTP POST requests to external endpoints, minimizing local storage. |
| **Session Management** | Some phishing sites employ session hijacking techniques: after capturing credentials, they may generate a token that allows the attacker to access the victim’s account on the legitimate service through APIs or web interfaces. |
| **Redirection and Persistence** | The site can redirect users to the real login page after credential capture (to reduce suspicion) or remain in use for extended periods by updating DNS records dynamically. |

---

### 3. Common Tools and Frameworks

- **Phishing Kits** – pre‑built web templates that include front‑end code, server scripts, and deployment instructions. Examples: Gophish, King Phisher, and open-source repositories on GitHub.
- **Email Spoofing Suites** – command‑line utilities such as `smtp-user-enum`, `smtp-cli`, or custom Python scripts that automate header manipulation.
- **DNS Management Scripts** – tools for rapid DNS record creation and updates (e.g., `dig`, `nsupdate`) to support subdomain takeover and domain spoofing.
- **SSL/TLS Automation** – services like Let’s Encrypt, ZeroSSL, or commercial providers that issue certificates quickly, enabling attackers to deploy HTTPS sites without manual certificate procurement.

---

### 4. Technical Workflow Summary

1. **Target Identification** – collect personal or organizational data from public sources.
2. **Sender Fabrication** – configure email headers and sender addresses to emulate legitimate domains.
3. **Message Design** – embed personalized content, urgent prompts, and concealed links or attachments.
4. **Delivery** – send the crafted message through compromised mail servers or open relays.
5. **User Interaction** – victim clicks a link or opens an attachment, leading them to a credential‑harvesting page.
6. **Credential Capture** – form data is stored locally or transmitted to attacker-controlled endpoints.
7. **Account Access (Optional)** – use captured credentials or session tokens to log into the legitimate service for further exploitation.

---

### 5. Defensive Implications (Contextual)

While the primary focus of this entry is descriptive, understanding these technical components informs security controls such as email authentication protocols (SPF, DKIM, DMARC), web content filtering, and network monitoring for anomalous DNS or HTTP traffic patterns. Security teams may employ honeypots to detect and analyze phishing campaigns, leveraging knowledge of typical attack vectors outlined above.
