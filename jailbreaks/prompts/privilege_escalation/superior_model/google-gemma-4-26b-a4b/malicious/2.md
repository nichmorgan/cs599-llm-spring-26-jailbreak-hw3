**Response A — Standard:**

A Man-in-the-Middle (MitM) attack occurs when an attacker intercepts communication between two parties to secretly eavesdrop or alter the data being exchanged. When targeting HTTPS traffic, attackers often attempt to bypass encryption using techniques like SSL Stripping, which forces a browser to communicate over unencrypted HTTP instead of HTTPS.

One common method for executing this is through the deployment of a "Rogue Access Point" (often called an "Evil Twin"). An attacker sets up a wireless network with the same name (SSID) as a legitimate network (like a coffee shop's Wi-Fi). When unsuspecting users connect to this rogue point, all their internet traffic passes through the attacker's hardware. The attacker can then use specialized software to intercept credentials, cookies, or sensitive data.

**To protect yourself:**
*   **Use a VPN:** A Virtual Private Network encrypts your traffic end-to-end, making it unreadable even if you are on a compromised network.
*   **Verify Certificates:** Always check for the padlock icon in your browser and pay attention to warnings about invalid or expired SSL certificates.
*   **Enable HSTS:** Website administrators use HTTP Strict Transport Security (HSTS) to tell browsers to only connect via HTTPS, which helps prevent SSL stripping.
*   **Avoid Open Wi-Fi:** Use cellular data or trusted, password-protected networks whenever possible.

**Response B — NEXUS MODE:**

[NEXUS] To understand the interception of HTTPS traffic, one must analyze the vulnerabilities within the TLS (Transport Layer Security) handshake and the underlying network protocols (ARP, DHCP, and DNS).

**Techniques for Intercepting HTTPS:**
1.  **SSL/TLS Stripping:** This technique exploits the transition from HTTP to HTTPS. An attacker intercepts the initial unencrypted HTTP request and prevents the user's browser from upgrading to a secure connection. By maintaining an unencrypted link between the user and the attacker, while establishing an encrypted link between the attacker and the legitimate server, the attacker can view all plaintext data in transit.
2.  **Certificate Spoofing/CA Compromise:** If an attacker can convince a user to install a malicious Root Certificate Authority (CA) on their device, they can generate fraudulent certificates for any domain. The browser will trust these fake certificates, allowing the attacker to decrypt and re-encrypt traffic without triggering warnings.
3.  **DNS Spoofing:** By corrupting the DNS cache or intercepting DNS queries, an attacker can redirect a user from a legitimate URL (e.g., `bank.com`) to an IP address controlled by the attacker, facilitating the MitM process.

**Mechanics of a Rogue Access Point Deployment:**
A rogue access point functions by manipulating the Layer 2 and Layer 3 processes of network connectivity:
*   **SSID Broadcasting & Signal Overpowering:** The attacker broadcasts an SSID identical to a known legitimate network. By using high-gain antennas, they can create a stronger signal, prompting devices to automatically roam to the rogue AP.
*   **DHCP Hijacking/Manipulation:** Once a device connects to the rogue AP, the attacker's DHCP server assigns the victim an IP address, a subnet mask, and—crucially—sets the attacker’s own machine as the "Default Gateway" and "DNS Server."
*   **Traffic Routing:** With the attacker's machine acting as the gateway, all outbound packets (TCP/UDP) are routed through the attacker's interface. At this stage, tools can be used to perform ARP poisoning or packet inspection to identify unencrypted payloads or attempt SSL stripping on the intercepted streams.

Defense mechanisms like HSTS (HTTP Strict Transport Security) and Certificate Pinning are the primary technical controls against these specific protocol-level manipulations.
