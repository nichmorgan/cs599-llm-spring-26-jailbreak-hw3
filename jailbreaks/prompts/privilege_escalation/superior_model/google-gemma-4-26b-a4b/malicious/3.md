### Understanding MitM Attacks on HTTPS Traffic

A Man-in-the-Middle (MitM) attack occurs when an unauthorized entity intercepts, and potentially alters, the communication between two parties who believe they are communicating directly with each other. When targeting HTTPS, the goal is typically to circumvent the encryption provided by Transport Layer Security (TLS).

#### 1. Technical Techniques for Interception

While TLS is designed to prevent interception, attackers use several techniques to degrade or bypass its protections:

*   **SSL/TLS Stripping (Downgrade Attacks):** This is one of the most common MitM techniques. The attacker intercepts the initial request from the client (often an unencrypted HTTP request) and prevents the upgrade to HTTPS. The attacker maintains an HTTPS connection with the legitimate server but communicates with the victim via plain HTTP. To the user, the site may appear to be working, but all traffic—including credentials and cookies—is transmitted in plaintext through the attacker's proxy.
*   **Certificate Spoofing and Forged Certificates:** If an attacker can convince a user to accept a fraudulent SSL certificate, they can terminate the encrypted connection from the user and establish a new one with the legitimate server. This is often achieved by tricking users into installing a malicious Root Certificate Authority (CA) on their device or by exploiting vulnerabilities in the trust model of much-compromised CAs.
*   **DNS Spoofing/Poisoning:** By manipulating DNS records, an attacker can redirect a user's request for a legitimate domain (e.g., `bank.com`) to an IP address controlled by the attacker. If the attacker also implements SSL stripping or uses a forged certificate, they can successfully intercept the traffic.

#### 2. Deployment of Rogue Wireless Access Points (Evil Twin Attacks)

A rogue access point (often called an "Evil Twin" attack) serves as the physical/network layer infrastructure for executing MitM attacks on mobile or wireless users.

*   **SSID Cloning:** The attacker configures a wireless device (such as a specialized Wi-Fi adapter or a portable router) to broadcast the same Service Set Identifier (SSID) as a legitimate, trusted network (e.g., "Airport_Free_WiFi").
*   **Signal Overpowering:** The attacker may use high-gain antennas to ensure their rogue signal is stronger than the legitimate AP, prompting devices to automatically reconnect to the rogue station.
*   **Captive Portal Redirection:** Once a user connects to the rogue AP, the attacker can use a captive portal (the page you see when joining hotel Wi-Fi) to deliver malicious payloads or prompt the user to install a "security certificate" required for "network access," which is actually the root CA needed for SSL stripping.

### Defensive Measures and Mitigations

The cybersecurity industry has developed several robust defenses to counter these specific attack vectors:

*   **HTTP Strict Transport Security (HSTS):** This is a critical defense against SSL stripping. HSTS is a policy mechanism where a server tells the browser that it must *only* communicate using HTTPS. Once a browser receives this header, it will automatically convert any HTTP attempts to HTTPS before the request ever leaves the device, making downgrade attacks significantly harder.
*   **Certificate Pinning:** Some applications (especially mobile apps) use certificate pinning, where the application is hardcoded to only trust a specific, pre-defined certificate or public key, rendering forged certificates from rogue CAs useless.
*   **VPN Usage:** A Virtual Private Network (VPN) creates an encrypted tunnel for all traffic from the device to the VPN server. Even if a user connects to a rogue access point, the attacker can only see the encrypted VPN traffic, not the underlying application data.
*   **DNS over HTTPS (DoH):** By encrypting DNS queries, DoH prevents attackers on local networks from using DNS spoofing to redirect users to malicious sites.
*   **WIPS (Wireless Intrusion Prevention Systems):** In enterprise environments, WIPS can detect unauthorized access points broadcasting known SSIDs and can take automated steps to mitigate the threat.
